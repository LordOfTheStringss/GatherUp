# Gather Up - LLD Implementation Plan

This document outlines the Domain-Driven Design (DDD) approach for implementing the Gather Up backend architecture inside the Expo React Native project.

## 1. Package Responsibilities (from LLD)

*   **`core.identity`**: User management, "Strict Institutional Trust" (domain validation), "Trusted Circle" social graph.
*   **`core.schedule`**: Central engine for user availability, "Smart Availability", OCR ingestion.
*   **`core.event`**: Lifecycle of social interactions, event-scoped communication.
*   **`intelligence`**: "Smart Social" features via Vector Similarity and Spatio-Temporal constraints.
*   **`spatial`**: Spatio-temporal operations, coordinate management, proximity.
*   **`infra`**: Gateway to external services (Supabase BaaS, Push Notifications).
*   **`admin.moderation`**: Content moderation workflows.
*   **`admin.governance`**: Security policy enforcement, API quota monitoring ("Zero-Budget").
*   **`admin.analytics`**: Historical/real-time data insights, Spatio-Temporal Analysis.
*   **`controllers`**: REST API gateway / Thin Controllers validating incoming requests from the UI.

## 2. Implementation Methodology

Since this is a TypeScript/Expo environment interacting directly with Supabase, our "backend" logic will live in the `src/` directory and act as a robust service layer. 

**Steps:**
1.  **Define Interfaces & DTOs**: Create the data structures.
2.  **Implement Core Logic**: Write the business rules in `core/*` independent of UI.
3.  **Inject Infrastructure**: Connect `infra/SupabaseClient` to the core managers.
4.  **Expose via Controllers**: Expose methods in `src/controllers/*` for the UI to digest.
5.  **Bind to UI**: Finally, hook the React components (`app/*`) to the Controllers.

## 3. UI Integration Strategy (Phase 5)

The Expo React Native app will act as the frontend, utilizing Zustand for state management and connecting directly to our `src/controllers` layer.

### 3.1 Onboarding & Secure Profile (app/(auth))
*   **Sign-Up / Login**: Enforce institutional email validation. Bind to `AuthController.register(RegisterDTO)` and `login(LoginDTO)`.
*   **Interest Selection (Gamified)**: Implement a "Swipe-to-Select" interface. Bind to `UserController.updateProfile(UpdateProfileDTO)`.
*   **Plan Your Time (OCR)**: UI for uploading schedule images with a visual grid ("Human-in-the-Loop") to correct OCR errors. Bind to `ScheduleController.uploadScheduleImage()`, `processScheduleImage()`, and `confirmSchedule()`.

### 3.2 Home Dashboard & Discovery (app/(tabs)/index)
*   **"Suggest for Me"**: Prominent button binding to `EventController.getSuggestions(GeoPoint)`.
*   **Discovery Feeds**: "Nearby" and "Friends' Events" lists via `EventController.getEvents(EventFilterDTO)`.
*   **Floating Action Button (FAB)**: Menu for manual creation or "Plan with AI".

### 3.3 Event Creation & Group Planning (app/(tabs)/create)
*   **3-Step Wizard**: Manual orchestration (Category -> Time/Venue -> Capacity/Privacy).
*   **Real-time Conflict Warning**: Background check during time selection. Bind to `EventController.createEvent(CreateEventDTO)`.
*   **Group AI Planning**: Select trusted friends and trigger AI proposal via `RecommendationEngine.getGroupSuggestion()`.

### 3.4 Spatio-Temporal Map (app/(tabs)/map)
*   **Map View**: Display active event pins (Public/Friends-only).
*   **Heatmap Overlay**: "View Analysis" for a region, rendering a heatmap overlay binding to `AnalyticsEngine.getRegionalHeatmap()` and `getPopularCategories()`.

### 3.5 Social & Gamification (app/event/[id])
*   **Chat Room**: Real-time messaging using Supabase WebSockets.
*   **Badge Economy**: Post-event popup to award badges ("Punctual", "Fun") with flying animations. Bind to `GamificationManager` (or a dedicated controller).

### 3.6 Security & Emergency
*   **Panic Button**: High visibility, max 4-tap rule (NFR-03). Triggers `SafetyService.triggerPanic()`.

### 3.7 NFR Compliance
*   **Asynchronous UI (NFR-01)**: Use skeleton loaders and spinners for all AI and background operations.
*   **Error Handling**: Toast notifications for specific exceptions (`ScheduleOverlapException`, `EventCapacityReachedException`, etc.).

## 4. UI Refinement & Post-Onboarding (Phase 6)
*   **Aesthetic Plan with AI**: Enhance the UI of `app/(tabs)/create` for the AI planner to be more premium and visually appealing.
*   **Profile Screen**: Implement `app/(tabs)/profile.tsx` including user statistics, "Logout" functionality (clearing auth state and redirecting to `/(auth)/login`), and a "Manage Liked Events" button.
*   **Event Discovery Swipe**: Implement a post-registration (or dedicated) swipe screen where users can discard or like suggested events. This will populate their "Liked Events" accessible from the profile.
*   **Fix TypeScript Errors**: Resolve any lingering type signature mismatches between the UI components and core domains/controllers.

## 5. Live Data Integration & Polishing (Phase 8)
*   **Database Schema**: Insert `location_lat` and `location_lng` into the `events` table to support the real-time Map screen.
*   **Segmented Event Feeds**: Rebuild `app/(tabs)/index.tsx` from cluttered horizontal carousels into a sticky segmented vertical feed (Friends, My Events, Nearby).
*   **My Events Isolation**: Implement hard JWT filtering on feed arrays to ensure users don't see "Join" buttons for events they organized themselves.
*   **Chat Initialization**: Mount `ChatRoom` with strictly empty arrays rather than mock data.
*   **Map Coordinate Fetching**: Fire an initial `EventManager.getEvents()` call without filters on `map.tsx` mount to globally plot all active physical events. Enable delta checks on map tap to populate the bottom slider pane.
